I donna suppose you could speed things up?
Written By Date written

How well do you know JavaScript and CSS? Without using jQuery or BootStrap? Yeah. For me, the answer is not good enough. This project is my attempt to work through the process of creating a "modern" looking design, without bloating my header with a huge chunk of CDN calls for minified files. I chose to limit myself to plain JavaScript and CSS 3. I leaned heavily on some StackOverflow articles (who doesn't) and several W3Schools examples (you don't?). Other than some gentle nudges in the right direction, I'm responsible for the structure and (sometimes) not-so-perfectly-optimized code. I'll post my source code to BitBucket or GitHub shortly. In short, this was a fun exercise.

I used Visual Studio Code for this project, rather than Visual Studio 2015. First, I wanted to develop familiarity with the program. It's layout is a little different and definitely more bare-bones than VS2015. I also wanted to make use of that simplicity for switching between files, making saves, and just keeping the file structure simple as possible. If you're like me, when you're navigating the unknown (and JavaScript sometimes falls into that category), keeping things simple reduces your search for bugs and other dumb errors we all make.

My goals for this project were simple:

  • Create a single-page application
  • Use only HTML, CSS 3, and JavaScript for any features of the application
  • Make the page responsive to screen-size
  • The navigation should have a collapsable menu for small screens and indicate the active section on all screens
  • Navigation should smoothly take you to the desired section
  • The navigation and header should be sticky on the top of the screen, no matter what content section is displayed
